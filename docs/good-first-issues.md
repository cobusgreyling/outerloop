## How to claim

1. Find an issue (or pick from the tables above).
2. Comment: **I'd like to work on this.**
3. Fork, branch (`feat/` or `docs/`), implement with tests.
4. Add a changeset if you change published package behavior.
5. Open a PR with rationale — we dogfood outer-loop answerability on our own merges.
