# Story: Dogfooding the release that fixed `npx`

**Date:** 2026-07-09  
**Context:** v0.3.1 adoption improvements  
**Outcome:** Success — blocker identified and fixed before broad promotion

## Situation

v0.3.0 shipped with docs telling adopters to run:

```bash
npx @cobusgreyling/outerloop init
```

An adoption review found the command exited silently — no `.outerloop/`, no error. Local `node packages/cli/dist/cli.js` worked; npm bin symlinks did not.

## Root cause

The CLI only called `run()` when `process.argv[1]` exactly matched `import.meta.url`. npm's `.bin/outerloop` symlink points at `dist/cli.js`, so the paths diverged and the CLI loaded without executing.

## Fix

Symlink-safe entrypoint check via `realpathSync`, plus a CI smoke test that `npm pack`s the CLI tarball and runs `npx outerloop init` against it.

## Lesson

**Test the published artifact, not just the monorepo.** Turbo build + vitest passed; consumers still got a no-op binary. The smoke test now gates every PR.

## Answerability chain (this repo)

For significant releases to `main`, maintainers should:

1. Package evidence for the release PR (`outerloop evidence package`)
2. Issue a verdict with rationale (`outerloop verdict issue`)
3. Tag release and verify `outerloop ledger why <tag>`

v0.3.1 is the first release validated with the published-package smoke test in CI.
